# 실시간 명령 시스템 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 플레이어 캐릭터에게 명령을 내리고 행동 시간을 시작하는 규칙을 정의한다

---

# 1. 문서 범위

본 문서는 전투 중 명령 가능 조건, 명령 단계, 명령 확정, 명령 취소를 정의한다.  
키보드와 마우스 입력 매핑은 `입력_조작_명세.md`에서 정의한다.  
플레이어 상태 전환은 `플레이어_상태_흐름.md`에서 정의한다.  
행동 효과와 스킬 규칙은 `플레이어_행동과_스킬.md`에서 정의한다.

---

# 2. 명령 가능 조건

플레이어 캐릭터는 아래 조건을 모두 만족할 때만 명령을 받을 수 있다.

- 플레이어 상태가 `명령 대기`이다
- HP가 0보다 크다
- 전투 전체 상태가 `전투 진행`이다
- 현재 선택 중인 다른 명령 확정 절차와 충돌하지 않는다

행동 중, 행동 발동, 전투 불능 상태의 캐릭터는 새 명령을 받을 수 없다.

---

# 3. 명령 단계

명령은 아래 단계로 진행한다.

| 단계 | 의미 | 다음 단계 |
|---|---|---|
| 캐릭터 선택 | 명령 가능한 캐릭터를 선택한다 | 행동 선택 |
| 행동 선택 | 사용할 행동을 선택한다 | 대상 선택 또는 명령 확정 |
| 대상 선택 | 행동 대상이 필요한 경우 대상을 고른다 | 명령 확정 |
| 명령 확정 | 비용과 대상 유효성을 확인하고 행동 시간을 시작한다 | 종료 |
| 명령 취소 | 이전 단계 또는 명령 대기로 돌아간다 | 이전 단계 |

대상이 고정된 행동은 대상 선택 단계를 생략할 수 있다.  
보스 1마리 대상 공격은 기본적으로 보스를 자동 대상으로 지정할 수 있다.

---

# 4. 명령 확정 처리

명령 확정 시 아래 순서로 처리한다.

1. 선택 캐릭터가 여전히 명령 가능한지 확인한다
2. 선택 행동이 해당 캐릭터에게 허용되는지 확인한다
3. MP와 기타 비용을 확인한다
4. 대상이 유효한지 확인한다
5. 비용을 적용한다
6. 캐릭터 상태를 `행동 중`으로 바꾼다
7. `current_action_id`, `current_target_ids`, `remaining_action_time_sec` 값을 설정한다
8. 명령 UI를 닫고 다음 명령 가능한 캐릭터를 찾는다

[확인 필요] 비용을 명령 확정 시점에 차감할지, 행동 발동 시점에 차감할지 정해야 한다.  
프로토타입 기본값은 `명령 확정 시점 차감`이다. 이 방식은 행동 예약 후 남은 MP 표시가 단순하다.

---

# 5. 명령 취소 처리

명령 취소는 현재 단계에 따라 다르게 처리한다.

| 현재 단계 | 취소 처리 |
|---|---|
| 캐릭터 선택 | 선택 포커스를 해제하거나 유지한다 |
| 행동 선택 | 캐릭터 선택 상태로 돌아간다 |
| 대상 선택 | 행동 선택 상태로 돌아간다 |
| 명령 확정 이후 | 취소할 수 없다 |

명령이 확정되기 전에는 캐릭터 상태를 `명령 선택 중`에서 `명령 대기`로 되돌릴 수 있다.  
명령 확정 이후 행동 취소는 상태 흐름과 행동 취소 효과에서 처리한다.

---

# 6. 명령 선택 중 시간 흐름

프로토타입 기본값은 `명령 선택 중에도 전투 시간이 흐른다`이다.  
따라서 명령 메뉴가 열린 상태에서도 다른 캐릭터와 보스의 행동 시간은 계속 감소한다.

전투 피로도가 높다면 테스트 옵션으로 아래 방식을 비교할 수 있다.

| 옵션 | 처리 | 영향 |
|---|---|---|
| 실시간 유지 | 명령 선택 중에도 시간이 흐른다 | Reverse ATB 감각이 강하다 |
| 시간 감속 | 명령 선택 중 전투 시간이 느려진다 | 조작 부담이 줄어든다 |
| 일시정지 | 명령 선택 중 전투 시간이 멈춘다 | 턴제 감각이 강해진다 |

프로토타입 구현 기본값은 실시간 유지로 둔다.

---

# 7. 자동 포커스 규칙

명령 가능한 캐릭터가 여러 명이면 명령 포커스는 아래 규칙을 따른다.

1. 현재 선택 캐릭터가 행동을 시작하면 다음 명령 대기 캐릭터로 이동한다
2. 새로 명령 대기 상태가 된 캐릭터는 후보 목록 뒤에 추가한다
3. 플레이어가 직접 캐릭터를 선택하면 자동 포커스보다 직접 선택을 우선한다
4. 명령 대기 캐릭터가 없으면 명령 메뉴를 열지 않는다

입력으로 포커스를 이동하는 방식은 `입력_조작_명세.md`를 따른다.

---

# 8. 명령 데이터

명령 처리 중에는 최소한 아래 값을 관리한다.

| 값 | 설명 |
|---|---|
| selected_character_id | 현재 명령 대상 캐릭터 |
| command_step | 현재 명령 단계 |
| selected_action_id | 선택된 행동 ID |
| selected_target_ids | 선택된 대상 ID 목록 |
| command_opened_time_sec | 명령 메뉴가 열린 전투 시각 |

확정된 명령은 플레이어 상태 값의 `current_action_id`, `current_target_ids`, `remaining_action_time_sec`로 옮긴다.

---

# 9. 예외 처리

| 상황 | 처리 |
|---|---|
| 선택 중인 캐릭터가 전투 불능이 됨 | 명령 메뉴를 닫고 포커스를 해제한다 |
| 선택 대상이 사라지거나 전투 불능이 됨 | 대상 선택 단계로 되돌린다 |
| 비용이 부족함 | 행동 선택 단계에서 확정을 막는다 |
| 행동 중인 캐릭터를 선택함 | 명령 메뉴는 열지 않고 상태 정보만 표시한다 |
| 승리 또는 패배 상태가 됨 | 명령 UI를 닫고 전투 입력을 막는다 |
