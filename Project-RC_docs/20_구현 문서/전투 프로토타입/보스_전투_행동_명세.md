# 보스 전투 행동 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 보스가 행동을 선택하고 예고, 진행, 발동하는 규칙을 정의한다

---

# 1. 문서 범위

본 문서는 보스 1마리의 행동 선택과 행동 발동 규칙을 다룬다.  
보스 상태 전환은 `보스_상태_흐름.md`에서 정의한다.  
전투 틱 안에서 보스 행동이 처리되는 순서는 `전투_전체_진행_흐름.md`에서 정의한다.  
기절 발생과 약점 갱신은 `속성_약점_기절_시스템.md`에서 정의한다.

일반 몬스터 다수전은 본 문서 범위가 아니다.

---

# 2. 보스 행동 루프

보스 행동은 아래 흐름으로 처리한다.

1. 보스가 행동 가능한 상태인지 확인한다
2. 사용 가능한 행동 후보를 만든다
3. 행동 조건과 가중치를 계산한다
4. 행동 하나를 선택한다
5. 대상을 결정한다
6. 예고가 필요한 행동이면 `행동 예고` 상태로 전환한다
7. 예고가 없거나 예고가 끝나면 `행동 중` 상태로 전환한다
8. 행동 시간이 끝나면 발동 이벤트를 생성한다
9. 효과 적용이 끝나면 보스를 `정상` 상태로 되돌린다

보스가 `기절` 또는 `전투 불능` 상태이면 신규 행동을 선택하지 않는다.

---

# 3. 행동 가능 조건

보스는 아래 조건을 모두 만족할 때만 새 행동을 선택한다.

- 보스 상태가 `정상`이다
- 보스 HP가 0보다 크다
- 플레이어 파티에 전투 가능한 대상이 1명 이상 있다
- 현재 진행 중인 보스 행동이 없다

플레이어 전원이 전투 불능이면 보스 행동을 새로 선택하지 않고 전투 전체 진행에서 패배 조건을 처리한다.

---

# 4. 행동 후보 생성

행동 후보는 보스 행동 데이터에서 가져온다.  
각 행동은 조건을 만족할 때만 후보에 들어간다.

| 조건 예시 | 의미 |
|---|---|
| normal | 기본 상태에서 사용 가능 |
| hp_below_ratio | 보스 HP 비율이 기준 이하일 때 사용 가능 |
| weakness_revealed | 약점이 공개된 상태에서 사용 가능 |
| player_low_hp | HP가 낮은 플레이어가 있을 때 사용 가능 |
| after_stun | 기절 종료 후 사용 가능 |

[확인 필요] `after_stun` 행동을 프로토타입에 넣을지 정해야 한다.  
기절 직후 강한 반격은 플레이어 보상감을 해칠 수 있으므로 기본 구현에서는 제외하는 것을 추천한다.

---

# 5. 가중치 선택

행동 후보가 2개 이상이면 가중치로 행동을 선택한다.

기본 계산 흐름은 아래와 같다.

```text
최종 가중치 = 기본 가중치 + 조건 보정값
```

프로토타입에서는 복잡한 AI 점수식보다 단순 가중치 선택을 우선한다.  
항상 최적 행동만 선택하면 패턴이 고정되고, 완전 랜덤은 대응 판단이 약해지기 때문이다.

[확인 필요] 보정값 계산식을 별도 데이터로 둘지, 코드 안의 임시 규칙으로 둘지 정해야 한다.

---

# 6. 대상 결정

보스 행동의 대상은 `target_rule`을 기준으로 결정한다.

| target_rule | 처리 |
|---|---|
| random_alive_player | 전투 가능한 플레이어 중 무작위 1명을 고른다 |
| lowest_hp_player | 현재 HP 비율이 가장 낮은 플레이어를 고른다 |
| highest_threat_player | 위협도 값이 가장 높은 플레이어를 고른다 |
| all_players | 전투 가능한 모든 플레이어를 대상으로 한다 |
| self | 보스 자신을 대상으로 한다 |

[확인 필요] 위협도 시스템을 프로토타입에 넣을지 정해야 한다.  
위협도 시스템이 없다면 `highest_threat_player`는 사용하지 않는다.

---

# 7. 예고와 발동

보스 행동은 예고가 필요한 행동과 즉시 진행 행동으로 나눈다.

| 구분 | 처리 |
|---|---|
| 예고 없음 | 선택 즉시 `행동 중` 상태가 되고 행동 시간이 진행된다 |
| 예고 있음 | `행동 예고` 상태에서 경고를 표시한 뒤 `행동 중`으로 전환한다 |

강한 전체 공격, 방해 행동, 약점 보호 행동은 예고를 사용하는 것을 추천한다.  
일반 단일 공격은 예고 없이 짧은 행동 시간만 사용할 수 있다.

---

# 8. 기절과 행동 취소

보스가 행동 예고 또는 행동 중일 때 기절하면 기본적으로 현재 행동을 취소한다.  
행동 취소 후 보스는 `기절` 상태가 된다.

이미 발동 이벤트로 수집된 행동은 전투 전체 진행의 동시 발동 규칙을 따른다.  
발동 이벤트가 수집되기 전이라면 기절로 취소한다.

---

# 9. 보스 행동 데이터

보스 행동은 최소한 아래 데이터를 사용한다.

| 필드 | 설명 |
|---|---|
| boss_action_id | 보스 행동 ID |
| display_name | 화면 표시 이름 |
| cast_time_sec | 행동 시간 또는 예고 시간 |
| target_rule | 대상 결정 규칙 |
| base_weight | 기본 선택 가중치 |
| condition | 사용 조건 |
| effect_tags | 효과 태그 |
| power | 기본 위력 |
| warning_level | 경고 수준 |

필드의 전체 정의는 `데이터_테이블_명세.md`를 따른다.

- [확인 필요] 보스 행동 가중치 보정식을 확정해야 한다
