# 플레이어 행동과 스킬 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 플레이어 행동의 분류, 실행 조건, 발동 처리, 예외 처리를 정의한다

---

# 1. 문서 범위

본 문서는 플레이어가 선택할 수 있는 행동의 구현 규칙을 다룬다.  
명령 가능 조건과 명령 확정은 `실시간_명령_시스템.md`에서 정의한다.  
플레이어 상태 전환은 `플레이어_상태_흐름.md`에서 정의한다.  
속성, 약점, 기절 판정은 `속성_약점_기절_시스템.md`에서 정의한다.  
구체적인 캐릭터별 스킬 목록과 샘플 수치는 콘텐츠 자료 문서에서 관리한다.

---

# 2. 행동 공통 규칙

모든 플레이어 행동은 아래 공통 흐름을 따른다.

1. 명령 단계에서 행동을 선택한다
2. 비용과 대상 유효성을 확인한다
3. 행동 시간이 시작된다
4. 행동 시간이 0 이하가 되면 발동 이벤트를 만든다
5. 발동 이벤트 처리 순서에 따라 효과를 적용한다
6. 효과 적용 후 캐릭터는 `명령 대기` 또는 `전투 불능` 상태가 된다

행동 시간이 있는 행동은 즉시 효과를 적용하지 않는다.

---

# 3. 행동 분류

프로토타입의 플레이어 행동은 아래 분류를 사용한다.

| 분류 | 역할 | 약점 판정 |
|---|---|---|
| 일반 공격 | 비용 없는 기본 피해 행동 | [확인 필요] |
| 방어 | 짧은 시간 안에 피해를 줄이거나 위험을 버틴다 | 없음 |
| 속성 공격 | 특정 속성으로 보스를 공격한다 | 가능 |
| 회복 | 아군 HP를 회복한다 | 없음 |
| 보조 | 강화, 약화, 행동 취소 같은 특수 효과를 적용한다 | 효과별 결정 |
| 아이템 | 소모품을 사용한다 | [보류] |

아이템은 프로토타입 기본 구현에서 제외하는 것을 추천한다.  
아이템을 넣으면 인벤토리, 소모 수량, 전투 외 데이터까지 함께 필요해진다.

---

# 4. 행동 시간 기준

행동 시간은 아래 기준 범위를 가안으로 사용한다.

| 구분 | 기준 범위 | 용도 |
|---|---|---|
| 짧음 | 1.0초 전후 | 방어, 약한 공격, 빠른 회복 |
| 보통 | 2.0~3.0초 | 주력 공격, 기본 스킬 |
| 김 | 4.0~5.0초 | 강한 피해, 큰 회복, 핵심 약점 공략 |

행동 시간은 Reverse ATB 전투에서 선택의 비용으로 사용한다.  
긴 행동은 더 큰 효과를 가져야 하지만, 발동 전 위험에 노출된다.

---

# 5. 비용 처리

MP 비용은 행동 데이터의 `mp_cost`를 사용한다.  
프로토타입 기본값은 명령 확정 시점에 비용을 차감하는 것이다.

| 상황 | 처리 |
|---|---|
| MP가 충분함 | 명령 확정 가능 |
| MP가 부족함 | 행동 선택은 가능하되 확정은 막는다 |
| 행동이 취소됨 | [확인 필요] 비용 환불 여부 결정 필요 |
| 행동 발동 전 캐릭터가 전투 불능이 됨 | 행동 취소 |

---

# 6. 대상 처리

행동의 대상은 `target_type`을 기준으로 결정한다.

| target_type | 처리 |
|---|---|
| self | 사용자 자신 |
| ally_single | 전투 가능한 아군 1명 |
| ally_all | 전투 가능한 모든 아군 |
| enemy_single | 적 1명 |
| enemy_all | 모든 적 |
| boss | 보스 1마리 |

보스 1마리 전투에서 `enemy_single`과 `boss`는 같은 대상으로 처리할 수 있다.  
다만 추후 다수전 확장을 고려하면 데이터 값은 구분해도 된다.

---

# 7. 효과 처리 순서

플레이어 행동이 발동되면 아래 순서로 처리한다.

1. 행동 주체가 전투 가능한 상태인지 확인한다
2. 대상이 유효한지 확인한다
3. 피해, 회복, 방어, 보조 효과를 적용한다
4. 속성이 있는 공격이면 약점 판정을 요청한다
5. 전투 로그와 UI 표시 이벤트를 생성한다
6. 승패 조건 확인은 전투 전체 진행으로 넘긴다

약점 판정은 피해 처리 이후에 요청한다.  
피해량이 0이어도 약점 판정을 할지는 `속성_약점_기절_시스템.md`에서 확정한다.

---

# 8. 방어 행동

방어 행동은 짧은 행동 시간으로 빠르게 발동되는 생존 행동이다.  
방어 효과는 발동 후 일정 시간 또는 다음 피격 1회에 적용할 수 있다.

프로토타입 기본값은 `다음 피격 1회 피해 감소`를 추천한다.

| 값 | 기본안 |
|---|---|
| 행동 시간 | 1.0초 |
| MP 비용 | 0 |
| 효과 | 다음 피격 피해 감소 |
| 지속 조건 | 피해를 1회 받거나 일정 시간이 지나면 종료 |

[확인 필요] 방어 효과가 행동 중인 캐릭터에게도 적용되는지 정해야 한다.

---

# 9. 예외 처리

| 상황 | 처리 |
|---|---|
| 행동 발동 직전 대상이 전투 불능이 됨 | 대상 선택 실패로 처리하고 효과를 적용하지 않는다 |
| 행동 발동 직전 보스가 전투 불능이 됨 | 행동을 폐기하고 승리 처리를 따른다 |
| 같은 틱에 여러 플레이어 행동이 발동됨 | 전투 전체 진행의 동시 발동 우선순위를 따른다 |
| 속성이 없는 행동이 보스에게 적중함 | 약점 판정을 요청하지 않는다 |
| 행동 중 cancel_cast를 받음 | 플레이어 상태 흐름의 취소 규칙을 따른다 |
