# 플레이어 상태 흐름 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 플레이어 캐릭터 1명의 전투 중 상태와 전환 조건을 정의한다

---

# 1. 문서 범위

본 문서는 플레이어 캐릭터의 개별 상태를 정의한다.  
전투 전체 진행 순서는 `전투_전체_진행_흐름.md`에서 정의한다.  
명령 단계와 입력 매핑은 `실시간_명령_시스템.md`, `입력_조작_명세.md`에서 정의한다.  
행동 효과와 스킬 규칙은 `플레이어_행동과_스킬.md`에서 정의한다.

---

# 2. 플레이어 상태 목록

플레이어 캐릭터는 아래 상태를 가진다.

| 상태 | 의미 | 입력 가능 여부 | 행동 시간 진행 여부 |
|---|---|---|---|
| 명령 대기 | 아무 행동도 하지 않고 명령을 기다린다 | 가능 | 없음 |
| 명령 선택 중 | 플레이어가 행동과 대상을 고르는 중이다 | 가능 | 없음 |
| 행동 중 | 명령이 확정되어 행동 시간이 진행 중이다 | 불가능 | 진행 |
| 행동 발동 | 행동 효과를 적용하는 순간이다 | 불가능 | 완료 |
| 전투 불능 | HP가 0 이하이거나 행동할 수 없는 상태다 | 불가능 | 정지 |

`명령 대기`는 기존 문서의 `대기`와 같은 의미다.  
문서 안에서는 입력 가능 상태임을 명확히 하기 위해 `명령 대기`로 표기한다.

---

# 3. 기본 전환

기본 전환은 아래 흐름을 따른다.

```text
명령 대기
→ 명령 선택 중
→ 행동 중
→ 행동 발동
→ 명령 대기
```

명령 선택을 취소하면 `명령 선택 중 → 명령 대기`로 돌아간다.  
행동 발동이 끝나면 해당 캐릭터는 다시 명령을 받을 수 있다.  
전투 불능이 되면 현재 상태와 관계없이 `전투 불능`으로 전환한다.

---

# 4. 상태별 진입 조건

| 상태 | 진입 조건 |
|---|---|
| 명령 대기 | 행동 발동이 끝났고 전투 불능이 아니다 |
| 명령 선택 중 | 명령 대기 상태의 캐릭터가 선택되었다 |
| 행동 중 | 행동, 대상, 비용 확인이 끝나고 명령이 확정되었다 |
| 행동 발동 | 남은 행동 시간이 0 이하가 되었다 |
| 전투 불능 | HP가 0 이하가 되었다 |

---

# 5. 상태별 종료 조건

| 상태 | 종료 조건 | 다음 상태 |
|---|---|---|
| 명령 대기 | 플레이어가 캐릭터를 선택한다 | 명령 선택 중 |
| 명령 선택 중 | 명령을 확정한다 | 행동 중 |
| 명령 선택 중 | 명령을 취소한다 | 명령 대기 |
| 행동 중 | 행동 시간이 끝난다 | 행동 발동 |
| 행동 발동 | 효과 적용이 끝난다 | 명령 대기 또는 전투 불능 |
| 전투 불능 | 부활 효과가 적용된다 | 명령 대기 |

[확인 필요] 프로토타입에서 부활 효과를 제공할지 정해야 한다.  
부활이 없다면 `전투 불능 → 명령 대기` 전환은 사용하지 않는다.

---

# 6. 행동 중 피격 처리

프로토타입 기본값은 `행동 중 피격을 받아도 행동 시간이 유지된다`이다.  
이 기본값은 플레이어가 긴 행동을 선택했을 때 결과를 예측하기 쉽다.

예외적으로 `cancel_cast` 효과를 가진 보스 행동은 플레이어의 행동을 취소할 수 있다.

| 상황 | 기본 처리 |
|---|---|
| 행동 중 일반 피해를 받음 | 행동 유지 |
| 행동 중 HP가 0 이하가 됨 | 행동 취소 후 전투 불능 |
| 행동 중 cancel_cast를 받음 | 행동 취소 후 명령 대기 |
| 행동 발동 틱에 HP가 0 이하가 됨 | 동시 발동 우선순위에 따라 처리 |

[확인 필요] 행동 취소 시 MP 비용을 환불할지 정해야 한다.

# 7. 플레이어 상태 값

플레이어 캐릭터는 전투 중 최소한 아래 값을 가진다.

| 값 | 설명 |
|---|---|
| character_id | 캐릭터 ID |
| hp | 현재 HP |
| mp | 현재 MP |
| state | 현재 플레이어 상태 |
| current_action_id | 진행 중인 행동 ID |
| current_target_ids | 현재 행동 대상 |
| remaining_action_time_sec | 남은 행동 시간 |
| is_command_focus | 현재 명령 포커스 여부 |

# 8. UI 표시 요구

플레이어 상태는 화면에서 아래처럼 읽혀야 한다.

| 상태 | 표시 요구 |
|---|---|
| 명령 대기 | 명령 가능 강조 표시 |
| 명령 선택 중 | 선택 캐릭터와 행동 메뉴 표시 |
| 행동 중 | 행동명과 남은 시간 표시 |
| 행동 발동 | 짧은 발동 연출 또는 로그 표시 |
| 전투 불능 | 조작 불가 상태 표시 |

상세 화면 배치는 `화면_구성_명세.md`를 따른다.
