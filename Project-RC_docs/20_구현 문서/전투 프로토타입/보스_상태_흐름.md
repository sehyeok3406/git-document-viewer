# 보스 상태 흐름 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 보스 1마리의 전투 중 상태와 전환 조건을 정의한다

---

# 1. 문서 범위

본 문서는 보스의 개별 상태를 정의한다.  
보스 행동 선택 규칙은 `보스_전투_행동_명세.md`에서 정의한다.  
약점 공개, 누적 공략, 기절 발생, 약점 갱신은 `속성_약점_기절_시스템.md`에서 정의한다.  
전투 전체 진행 순서는 `전투_전체_진행_흐름.md`에서 정의한다.

---

# 2. 보스 상태 목록

보스는 아래 상태를 가진다.

| 상태 | 의미 | 신규 행동 시작 여부 | 행동 시간 진행 여부 |
|---|---|---|---|
| 정상 | 기본 행동과 약점 판정이 모두 가능하다 | 가능 | 없음 |
| 행동 예고 | 강한 행동이나 특정 패턴을 준비 중이다 | 불가능 | 진행 |
| 행동 중 | 보스 행동 시간이 진행 중이다 | 불가능 | 진행 |
| 기절 | 행동이 제한되고 플레이어 보상 타이밍이 열린다 | 불가능 | 정지 |
| 전투 불능 | HP가 0 이하가 되어 전투에서 패배한 상태다 | 불가능 | 정지 |

보스의 일반 행동은 `행동 중`으로 처리한다.  
플레이어에게 미리 알려야 하는 강한 행동은 `행동 예고`를 거친 뒤 발동한다.

---

# 3. 기본 전환

기본 전환은 아래 흐름을 따른다.

```text
정상
→ 행동 예고 또는 행동 중
→ 정상
```

약점 누적 조건을 만족하면 현재 상태와 관계없이 `기절`로 전환할 수 있다.  
보스 HP가 0 이하가 되면 현재 상태와 관계없이 `전투 불능`으로 전환한다.

---

# 4. 상태별 진입 조건

| 상태 | 진입 조건 |
|---|---|
| 정상 | 보스 행동이 끝났고 기절 상태가 아니다 |
| 행동 예고 | 예고가 필요한 보스 행동이 선택되었다 |
| 행동 중 | 예고가 없는 행동이 선택되었거나 예고 시간이 끝났다 |
| 기절 | 공개 약점의 누적 공략 횟수가 요구치에 도달했다 |
| 전투 불능 | 보스 HP가 0 이하가 되었다 |

---

# 5. 상태별 종료 조건

| 상태 | 종료 조건 | 다음 상태 |
|---|---|---|
| 정상 | 보스 행동 선택 조건을 만족한다 | 행동 예고 또는 행동 중 |
| 행동 예고 | 예고 시간이 끝난다 | 행동 중 |
| 행동 중 | 행동 시간이 끝나고 효과 적용이 끝난다 | 정상 |
| 기절 | 기절 지속 시간이 끝난다 | 정상 |
| 전투 불능 | 없음 | 없음 |

---

# 6. 기절 중 처리

프로토타입 기본값은 `보스가 행동 중일 때 기절하면 해당 행동을 취소한다`이다.  
이 기본값은 약점 공략의 보상감을 가장 명확하게 보여준다.

| 상황 | 기본 처리 |
|---|---|
| 정상 상태에서 기절 발생 | 즉시 기절 |
| 행동 예고 중 기절 발생 | 예고 취소 후 기절 |
| 행동 중 기절 발생 | 행동 취소 후 기절 |
| 전투 불능과 기절이 동시에 발생 | 전투 불능 우선 |

[확인 필요] 이미 발동된 투사체나 지속 효과가 기절로 사라지는지 정해야 한다.

---

# 7. 보스 상태 값

보스는 전투 중 최소한 아래 값을 가진다.

| 값 | 설명 |
|---|---|
| boss_id | 보스 ID |
| hp | 현재 HP |
| state | 현재 보스 상태 |
| current_action_id | 진행 중인 보스 행동 ID |
| current_target_ids | 현재 행동 대상 |
| remaining_action_time_sec | 남은 행동 또는 예고 시간 |
| remaining_stun_time_sec | 남은 기절 시간 |

약점 슬롯 상태는 `속성_약점_기절_시스템.md`에서 별도로 관리한다.

---

# 8. UI 표시 요구

보스 상태는 화면에서 아래처럼 읽혀야 한다.

| 상태 | 표시 요구 |
|---|---|
| 정상 | 기본 HP와 약점 슬롯 표시 |
| 행동 예고 | 행동명, 대상, 위험도 표시 |
| 행동 중 | 행동 진행 또는 남은 시간 표시 |
| 기절 | 기절 강조, 남은 시간 표시 |
| 전투 불능 | 승리 전환 표시 |

상세 화면 배치는 `화면_구성_명세.md`를 따른다.

---

# 9. 확인 필요 사항

- [확인 필요] 강한 행동만 행동 예고를 사용할지, 모든 보스 행동에 예고 시간을 둘지 정해야 한다
- [확인 필요] 기절로 이미 발동된 지속 효과까지 제거할지 정해야 한다
- [확인 필요] 기절 종료 직후 보스가 즉시 행동을 선택할지, 짧은 복귀 지연을 둘지 정해야 한다
