# 전투 전체 진행 흐름 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 전투 시작부터 종료까지의 처리 순서와 시스템 간 호출 순서를 정의한다

---

# 1. 문서 범위

본 문서는 전투 전체 상태와 전투 진행 틱의 처리 순서를 다룬다.  
플레이어 개별 상태는 `플레이어_상태_흐름.md`에서 정의한다.  
보스 개별 상태는 `보스_상태_흐름.md`에서 정의한다.  
명령 입력 규칙은 `실시간_명령_시스템.md`와 `입력_조작_명세.md`에서 정의한다.  
약점 판정과 기절 처리는 `속성_약점_기절_시스템.md`에서 정의한다.

이 문서는 개별 행동 효과나 보스 AI 세부값을 확정하지 않는다.  
전투 루프 안에서 어떤 시스템이 어떤 순서로 실행되는지를 기준으로 삼는다.

---

# 2. 전투 전체 상태

전투는 아래 전체 상태를 가진다.

| 상태 | 의미 | 다음 상태 |
|---|---|---|
| 전투 준비 | 파티, 보스, 약점, UI, 계측 값을 초기화한다 | 전투 시작 |
| 전투 시작 | 시작 연출 또는 안내를 처리한다 | 전투 진행 |
| 전투 진행 | 명령, 행동 시간, 보스 행동, 약점 판정을 처리한다 | 승리 또는 패배 |
| 승리 | 보스 HP가 0 이하가 된 상태다 | 결과 |
| 패배 | 플레이어 전원이 전투 불능이 된 상태다 | 결과 |
| 결과 | 전투 결과와 다음 입력을 표시한다 | 전투 종료 또는 다음 화면 |

전투 전체 상태는 전투 씬의 최상위 상태다.  
플레이어와 보스의 개별 상태는 `전투 진행` 상태 안에서만 갱신한다.

---

# 3. 전투 준비 처리

전투 준비 상태에서는 아래 순서로 초기화한다.

1. 전투 설정 데이터를 읽는다
2. 플레이어 파티 3명의 전투 상태를 생성한다
3. 보스 전투 상태를 생성한다
4. 보스 약점 슬롯을 생성한다
5. 명령 시스템의 선택 포커스를 초기화한다
6. 전투 로그와 계측 값을 초기화한다
7. 화면 UI의 초기 표시 값을 만든다

전투 준비가 끝나면 `전투 시작` 상태로 전환한다.

---

# 4. 전투 진행 틱

전투 진행 상태에서는 매 프레임 또는 고정 갱신 단위마다 아래 순서로 처리한다.

1. 경과 시간을 계산한다
2. 입력을 수집한다
3. 플레이어 명령을 확정하거나 취소한다
4. 플레이어 행동 시간을 갱신한다
5. 보스 행동 선택과 행동 시간을 갱신한다
6. 발동 가능한 행동 이벤트를 수집한다
7. 발동 이벤트의 처리 순서를 결정한다
8. 피해, 회복, 방어, 상태 변화를 적용한다
9. 속성, 약점, 기절 판정을 처리한다
10. 전투 불능과 기절 종료 같은 상태 전환을 처리한다
11. 승리 또는 패배 조건을 확인한다
12. 전투 로그와 계측 값을 기록한다
13. UI 표시 값을 갱신한다

승리 또는 패배 조건이 충족되면 진행 중인 행동 시간과 입력 처리를 정지한다.

---

# 5. 발동 이벤트

행동 시간이 끝난 행동은 즉시 효과를 적용하지 않고 발동 이벤트로 수집한다.  
한 틱 안에서 여러 행동이 동시에 끝날 수 있기 때문이다.

발동 이벤트는 최소한 아래 값을 가진다.

| 필드 | 설명 |
|---|---|
| event_id | 발동 이벤트 임시 ID |
| actor_type | player 또는 boss |
| actor_id | 행동 주체 ID |
| action_id | 발동할 행동 ID |
| target_ids | 대상 ID 목록 |
| element_id | 속성 ID. 속성이 없으면 none |
| resolved_time_sec | 발동 시간이 된 전투 시각 |
| priority_group | 동시 발동 처리 우선순위 |

발동 이벤트의 상세 효과는 행동 데이터와 각 시스템 문서를 기준으로 처리한다.

---

# 6. 동시 발동 우선순위

동시에 여러 행동이 발동하면 아래 우선순위를 가안으로 사용한다.

1. 플레이어 방어 또는 회복 행동
2. 플레이어 보조 행동
3. 플레이어 공격 또는 약점 판정 행동
4. 보스 행동
5. 지속 피해 또는 지속 회복 효과

[확인 필요] 보스의 강한 예고 행동을 플레이어 공격보다 먼저 처리할지 정해야 한다.  
프로토타입 기본값은 플레이어 행동을 먼저 처리하는 것이다.  
이 기본값은 플레이어가 시전 완료 타이밍을 맞췄을 때 보상을 체감하기 쉽다.

---

# 7. 승패 조건

## 7-1. 승리 조건

보스 HP가 0 이하가 되면 승리한다.  
승리 상태로 전환하면 진행 중인 행동 이벤트는 더 이상 처리하지 않는다.

## 7-2. 패배 조건

플레이어 캐릭터 3명이 모두 전투 불능이면 패배한다.  
패배 상태로 전환하면 보스 행동과 플레이어 입력을 모두 정지한다.

# 9. 확인 필요 사항

- [확인 필요] 고정 갱신 단위를 사용할지, 프레임 시간 기반 갱신을 사용할지 정해야 한다
- [확인 필요] 동시 발동 우선순위를 최종 확정해야 한다
- [확인 필요] 승리 또는 패배가 발생한 틱에서 이미 수집된 이벤트를 폐기할지, 일부 표시할지 정해야 한다
- [확인 필요] 전투 결과 화면으로 넘어가는 입력 조건을 정해야 한다
