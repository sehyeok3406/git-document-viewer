# 전투 데이터 테이블 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 전투 프로토타입에 필요한 작성 데이터와 필드를 정의한다

---

# 1. 문서 범위

본 문서는 기획자가 작성하고 구현자가 읽을 전투 데이터 필드를 정의한다.  
런타임 상태 값은 각 상태 흐름 문서에서 정의한다.  
전투 로그와 테스트 계측 값은 `전투_로그_계측_명세.md`에서 정의한다.

---

# 2. 공통 작성 규칙

- ID는 영어 소문자와 언더스코어를 사용한다
- 표시명은 한국어를 사용할 수 있다
- 수치가 미정이면 `TBD`로 표시한다
- 프로토타입 임시값은 `[가안]`으로 표시한다
- 구현 필드와 기획 메모 필드를 구분한다

예시:

| id | name | note |
|---|---|---|
| hero_guardian | 수호자 | [가안] 방어형 캐릭터 |

---

# 3. 데이터 종류

전투 프로토타입은 아래 데이터를 사용한다.

| 데이터 | 역할 |
|---|---|
| 캐릭터 테이블 | 플레이어 캐릭터 기본 정보 |
| 행동 / 스킬 테이블 | 플레이어가 선택하는 행동 정보 |
| 속성 테이블 | 속성 표시와 식별자 |
| 보스 테이블 | 보스 기본 정보 |
| 약점 슬롯 테이블 | 보스 약점 구성 |
| 보스 행동 테이블 | 보스 행동 후보와 선택 조건 |
| 전투 설정 테이블 | 전투 1개의 파티, 보스, 승패 조건 |

---

# 4. 캐릭터 테이블

플레이어 캐릭터 기본 정보를 정의한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| character_id | 캐릭터 고유 ID | hero_01 |
| display_name | 화면 표시 이름 | 수호자 |
| role | 전투 역할 | 공격, 회복, 보조 |
| max_hp | 최대 HP | 120 |
| max_mp | 최대 MP | 40 |
| default_action_ids | 기본 행동 ID 목록 | attack, guard |
| skill_ids | 사용 가능한 스킬 ID 목록 | fire_slash |
| element_tags | 캐릭터가 주로 사용하는 속성 | fire |
| notes | 기획 메모 | 초안 |

---

# 5. 행동 / 스킬 테이블

플레이어가 선택할 수 있는 행동과 스킬을 정의한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| action_id | 행동 고유 ID | fire_slash |
| display_name | 화면 표시 이름 | 화염 베기 |
| action_type | 행동 구분 | attack, skill, guard, heal |
| element_id | 속성 ID. 속성이 없으면 none | fire |
| cast_time_sec | 행동 시간 | 2.5 |
| mp_cost | MP 비용 | 10 |
| target_type | 대상 | boss |
| power | 기본 위력 | 100 |
| weakness_check | 약점 판정 여부 | true |
| effect_tags | 효과 태그 | damage, reveal_weakness |
| ui_icon | UI 아이콘 키 | icon_fire |
| description | 화면 또는 기획 설명 | 화염 속성 공격 |
| notes | 기획 메모 | 초안 |

## 5-1. action_type 후보

- attack
- skill
- guard
- heal
- support
- item

`item`은 프로토타입 기본 구현에서 [보류]로 둔다.

## 5-2. target_type 후보

- self
- ally_single
- ally_all
- enemy_single
- enemy_all
- boss

## 5-3. effect_tags 후보

- damage
- heal
- guard
- buff
- debuff
- reveal_weakness
- stack_weakness
- stun_bonus
- delay
- cancel_cast

# 6. 속성 테이블

전투에서 사용하는 속성을 정의한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| element_id | 속성 ID | fire |
| display_name | 표시명 | 화염 |
| ui_icon | 아이콘 키 | icon_fire |
| ui_color | UI 색상 | TBD |
| description | 속성 설명 | 화염 계열 |
| notes | 기획 메모 | 초안 |

초기 속성 후보는 아래 6종이다.

| 표시명 | element_id 가안 |
|---|---|
| 물리 | physical |
| 화염 | fire |
| 방사능 | radiation |
| 중력 | gravity |
| 신성 | holy |
| 공허 | void |

[확인 필요] 속성 ID의 영어 표기를 최종 확정해야 한다.

# 7. 보스 테이블

| 필드 | 설명 | 예시 |
|---|---|---|
| boss_id | 보스 고유 ID | boss_proto_01 |
| display_name | 표시명 | 실험체 |
| max_hp | 최대 HP | 1000 |
| weakness_slot_count | 시작 약점 슬롯 수 | 2 |

| weakness_required_hits | 기본 기절 요구 공략 횟수 | 3 |
| stun_duration_sec | 기절 지속 시간 | 5 |
| action_pattern_id | 보스 행동 패턴 ID | boss_proto_pattern |
| phase_ids | 페이즈 ID 목록 | TBD |
| notes | 기획 메모 | 초안 |

# 8. 약점 슬롯 테이블

보스의 시작 약점 슬롯을 정의한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| weakness_slot_id | 약점 슬롯 ID | boss_proto_weak_01 |
| boss_id | 대상 보스 ID | boss_proto_01 |
| element_id | 약점 속성 | fire |
| initial_state | 시작 상태 | hidden |
| required_hits | 기절 요구 횟수 | 3 |
| can_repeat_after_disabled | 비활성화 후 재등장 가능 여부 | false |
| notes | 기획 메모 | 초안 |

## 8-1. initial_state 후보

- hidden
- revealed
- disabled

# 9. 보스 행동 테이블

보스가 사용할 행동과 선택 조건을 정의한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| boss_action_id | 보스 행동 ID | boss_claw |
| display_name | 표시명 | 할퀴기 |
| cast_time_sec | 행동 시간 또는 예고 시간 | 2.0 |
| target_rule | 대상 규칙 | random_alive_player |
| base_weight | 기본 선택 가중치 | 100 |
| condition | 사용 조건 | normal |
| effect_tags | 효과 태그 | damage |
| power | 기본 위력 | 80 |
| warning_level | 경고 수준 | low |
| notes | 기획 메모 | 초안 |

## 9-1. target_rule 후보

- random_alive_player
- lowest_hp_player
- highest_threat_player
- all_players
- self

## 9-2. warning_level 후보

- none
- low
- medium
- high
