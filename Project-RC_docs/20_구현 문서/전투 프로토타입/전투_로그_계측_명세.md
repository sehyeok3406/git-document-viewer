# 전투 로그 / 계측 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 프로토타입 전투 테스트에서 기록할 이벤트와 측정 값을 정의한다

---

# 1. 문서 범위

본 문서는 전투 중 화면 로그와 테스트 계측 값을 정의한다.  
화면에 표시되는 로그 위치와 줄 수는 `화면_구성_명세.md`에서 정의한다.  
전투 진행 틱에서 로그가 기록되는 시점은 `전투_전체_진행_흐름.md`에서 정의한다.

---

# 2. 화면 로그 이벤트

전투 화면 로그에는 아래 이벤트만 간결하게 표시한다.

| 이벤트 | 표시 내용 |
|---|---|
| action_started | 캐릭터 또는 보스가 행동을 시작했다 |
| action_resolved | 행동이 발동했다 |
| damage_applied | 피해가 적용되었다 |
| heal_applied | 회복이 적용되었다 |
| weakness_revealed | 약점이 공개되었다 |
| weakness_hit | 약점 누적 횟수가 증가했다 |
| weakness_miss | 약점이 아닌 속성으로 공격했다 |
| boss_stunned | 보스가 기절했다 |
| character_ko | 플레이어 캐릭터가 전투 불능이 되었다 |
| battle_finished | 승리 또는 패배가 확정되었다 |

화면 로그는 판단 보조 정보다.  
핵심 판단 정보는 HUD와 약점 UI에 먼저 표시한다.

---

# 3. 계측 값

프로토타입 테스트 중 아래 값을 수집한다.

| 값 | 설명 |
|---|---|
| battle_duration_sec | 전투 종료까지 걸린 시간 |
| command_count | 플레이어 명령 확정 횟수 |
| average_idle_time_sec | 캐릭터가 명령 대기 상태로 남은 평균 시간 |
| first_weakness_reveal_time_sec | 첫 약점 공개까지 걸린 시간 |
| stun_count | 기절 발생 횟수 |
| player_ko_count | 전투 불능 발생 횟수 |
| weakness_miss_count | 약점이 아닌 속성 공격 횟수 |
| victory_result | 승리 또는 패배 결과 |

---

# 4. 기록 시점

| 시점 | 기록 |
|---|---|
| 전투 시작 | 시작 시각, 파티, 보스 ID |
| 명령 확정 | command_count 증가 |
| 캐릭터가 명령 대기 진입 | idle 시작 시각 기록 |
| 캐릭터가 행동 중 진입 | idle 시간 누적 |
| 약점 최초 공개 | first_weakness_reveal_time_sec 기록 |
| 보스 기절 | stun_count 증가 |
| 플레이어 전투 불능 | player_ko_count 증가 |
| 전투 종료 | battle_duration_sec, victory_result 기록 |

---

# 5. 출력 형식

프로토타입 기본값은 디버그 출력 또는 메모리 내 임시 구조로 둔다.  
파일 저장이 필요해지면 CSV 또는 JSON 중 하나를 선택한다.
