# 전투 화면 구성 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 전투 UI의 화면 영역, 표시 정보, 갱신 조건을 구현 가능한 수준으로 정의한다

---

# 1. 문서 범위

본 문서는 전투 프로토타입 화면에 표시할 영역과 정보를 정의한다.  
UI/UX 방향성은 `../../10_게임 개요/전투 프로토타입/전투_UI_UX_기획.md`를 기준으로 한다.  
플레이어 상태 표시는 `플레이어_상태_흐름.md`를 따른다.  
보스 상태 표시는 `보스_상태_흐름.md`를 따른다.  
약점 표시는 `속성_약점_기절_시스템.md`를 따른다.

---

# 2. 화면 영역

프로토타입 전투 화면은 아래 영역으로 나눈다.

| 영역 | 위치 | 역할 |
|---|---|---|
| 보스 영역 | 화면 상단 또는 중앙 상단 | 보스, HP, 상태, 행동 예고 표시 |
| 약점 영역 | 보스 근처 | 약점 슬롯과 누적 공략 상태 표시 |
| 전투 공간 | 화면 중앙 | 캐릭터와 보스 스프라이트 표시 |
| 파티 상태 영역 | 화면 하단 | 플레이어 3명의 HP, MP, 상태, 행동 시간 표시 |
| 명령 메뉴 영역 | 하단 또는 선택 캐릭터 근처 | 행동 선택 |
| 타겟 선택 표시 | 대상 위 | 현재 선택 대상 표시 |
| 전투 로그 영역 | 측면 또는 하단 일부 | 주요 결과 메시지 표시 |

---

# 3. 정보 우선순위

전투 중 가장 먼저 읽혀야 하는 정보는 아래 순서다.

1. 명령 가능한 캐릭터
2. 현재 행동 중인 캐릭터와 남은 시간
3. 보스 약점 상태
4. 보스 기절 상태
5. 보스 행동 예고
6. HP/MP 변화
7. 전투 로그

전투 로그는 보조 정보다.  
핵심 판단 정보는 캐릭터 UI, 약점 UI, 보스 상태 UI에서 직접 읽혀야 한다.

---

# 4. 보스 영역

보스 영역에는 아래 정보를 표시한다.

- 보스 이름
- 보스 HP 바
- 현재 보스 상태
- 기절 남은 시간
- 현재 또는 다음 행동 예고
- 보스 행동 대상

보스가 `행동 예고` 상태이면 행동명, 대상, 위험도를 표시한다.  
보스가 `기절` 상태이면 텍스트, 색상, 애니메이션 중 최소 2가지 신호로 표현한다.

---

# 5. 약점 영역

약점 영역에는 아래 정보를 표시한다.

- 미공개 약점 슬롯
- 공개 약점 속성
- 약점 누적 공략 카운트
- 비활성화된 약점

표시 예시는 아래와 같다.

```text
약점: [?] [화염 2/3] [비활성]
```

약점이 새로 공개되면 짧은 강조 연출을 사용한다.  
기절 조건에 도달할 수 있는 약점은 다른 약점보다 더 눈에 띄게 표시할 수 있다.

---

# 6. 파티 상태 영역

파티 상태 영역에는 3명의 캐릭터 정보를 고정된 위치에 표시한다.

각 캐릭터 UI에 필요한 정보는 아래와 같다.

- 캐릭터 이름
- HP
- MP
- 현재 상태
- 현재 행동명
- 남은 행동 시간
- 명령 가능 강조

상태 표시 예시는 아래와 같다.

```text
1. 수호자   HP 120/120  MP 20/30  [명령 대기]
2. 화염술사 HP 80/80    MP 40/50  [화염구 1.7s]
3. 사제     HP 70/90    MP 30/60  [명령 대기]
```

---

# 7. 명령 메뉴 영역

명령 메뉴는 캐릭터가 `명령 선택 중` 상태일 때 열린다.

행동 항목에는 아래 정보를 표시한다.

- 행동명
- 속성
- 행동 시간
- 비용
- 대상 종류

예시:

```text
수호자 명령
> 공격        물리  1.5s  MP 0
  방어        없음  1.0s  MP 0
  도발        없음  2.0s  MP 5
```

MP가 부족한 행동은 선택은 가능하되 확정 불가 상태로 표시한다.

---

# 8. 타겟 선택 표시

타겟 선택 중에는 선택 가능한 대상과 현재 선택 대상을 구분한다.

- 선택 가능 대상: 약한 강조
- 현재 선택 대상: 강한 강조
- 선택 불가 대상: 흐림 처리

보스 1마리 전투에서는 적 대상 선택이 단순할 수 있다.  
회복이나 보조 행동이 있으면 아군 타겟 선택 UI가 필요하다.

---

# 9. 전투 로그 영역

전투 로그에는 `전투_로그_계측_명세.md`의 화면 로그 이벤트만 표시한다.  
프로토타입에서는 최근 3~5줄만 표시한다.

로그는 아래 이벤트를 우선 표시한다.

- 행동 발동
- 피해 또는 회복 결과
- 약점 공개
- 약점 누적
- 보스 기절
- 승리 또는 패배

---

# 10. 화면 상태별 표시

| 화면 상태 | 표시 기준 |
|---|---|
| 전투 시작 | 파티, 보스, 약점 슬롯 초기 표시 |
| 명령 대기 | 명령 가능한 캐릭터 강조 |
| 명령 선택 | 행동 메뉴 표시 |
| 행동 중 | 캐릭터 타이머 진행 표시 |
| 약점 공개 | 약점 슬롯 강조 |
| 보스 행동 예고 | 행동명, 대상, 위험도 표시 |
| 보스 기절 | 보스 영역 강조, 남은 시간 표시 |
| 승리/패배 | 결과 메시지와 다음 입력 표시 |

- [확인 필요] 전투 로그를 화면에 계속 둘지, 임시 테스트 UI로만 둘지 정해야 한다
