# 전투 입력 / 조작 명세서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 키보드와 마우스 입력이 전투 명령 단계에 어떻게 연결되는지 정의한다

---

# 1. 문서 범위

본 문서는 물리 입력과 UI 조작을 정의한다.  
명령 가능 조건과 명령 확정 규칙은 `실시간_명령_시스템.md`에서 정의한다.  
플레이어 상태 전환은 `플레이어_상태_흐름.md`에서 정의한다.  
화면 영역과 표시 정보는 `화면_구성_명세.md`에서 정의한다.

---

# 2. 입력 우선순위

프로토타입 기본 입력 우선순위는 아래와 같다.

1. 키보드
2. 마우스
3. 패드

키보드는 반복 테스트의 기본 입력으로 사용한다.  
마우스는 화면 요소를 직접 선택하는 보조 입력으로 사용한다.  
패드는 프로토타입 기본 범위에서 제외하고 `[보류]`로 둔다.

---

# 3. 기본 조작 흐름

전투 중 명령 입력은 아래 흐름을 따른다.

1. 명령 가능한 캐릭터를 선택한다
2. 행동 메뉴에서 행동을 선택한다
3. 필요한 경우 대상을 선택한다
4. 선택을 확정한다
5. 캐릭터가 행동 중 상태로 전환된다

취소 입력을 받으면 현재 명령 단계의 이전 단계로 돌아간다.

---

# 4. 키보드 조작

| 입력 | 동작 |
|---|---|
| 1 | 1번 캐릭터 선택 |
| 2 | 2번 캐릭터 선택 |
| 3 | 3번 캐릭터 선택 |
| ↑ / ↓ | 행동 메뉴 항목 이동 |
| ← / → | 타겟 또는 선택 영역 이동 |
| Enter | 현재 선택 확정 |
| Esc | 취소 또는 이전 단계 |
| Space | 다음 명령 가능한 캐릭터로 이동 |

행동 중인 캐릭터를 숫자키로 선택하면 명령 메뉴를 열지 않고 상태 정보만 표시한다.

---

# 5. 마우스 조작

| 입력 | 동작 |
|---|---|
| 캐릭터 UI 클릭 | 해당 캐릭터 선택 |
| 행동 버튼 클릭 | 행동 선택 |
| 보스 클릭 | 적 대상 선택 |
| 아군 캐릭터 클릭 | 아군 대상 선택 |
| 우클릭 | 취소 또는 이전 단계 |

마우스 입력은 키보드 입력과 같은 명령 단계로 연결한다.  
마우스로 선택한 캐릭터나 대상은 키보드 포커스에도 반영한다.

---

# 6. 선택 가능 / 불가능 처리

| 대상 | 입력 처리 |
|---|---|
| 명령 대기 캐릭터 | 선택하고 명령 메뉴를 연다 |
| 행동 중 캐릭터 | 상태 정보만 표시한다 |
| 전투 불능 캐릭터 | 선택 불가 표시를 낸다 |
| 살아 있는 보스 | 적 대상 행동의 대상으로 선택한다 |
| 전투 불능 보스 | 대상 선택을 막는다 |

선택 불가능한 요소를 클릭하거나 키보드로 확정하면 명령 단계는 변하지 않는다.

---

# 7. 자동 선택 보조

대기 중인 캐릭터가 여러 명일 때 자동 선택 보조는 아래 규칙을 따른다.

- 현재 선택 캐릭터가 행동을 시작하면 다음 명령 대기 캐릭터로 이동한다
- `Space` 입력은 명령 대기 캐릭터만 순환한다
- 명령 대기 캐릭터가 없으면 포커스를 유지하지 않는다
- 직접 숫자키 또는 클릭으로 선택한 캐릭터는 자동 선택보다 우선한다

자동 선택 후보의 관리 규칙은 `실시간_명령_시스템.md`를 따른다.

---

# 8. 입력 예외 처리

| 상황 | 처리 |
|---|---|
| 메뉴가 열린 상태에서 캐릭터가 전투 불능이 됨 | 메뉴를 닫고 포커스를 해제한다 |
| 대상 선택 중 대상이 사라짐 | 대상 선택을 다시 요구한다 |
| 승리 또는 패배 상태 | 전투 입력을 막고 결과 입력만 받는다 |
| 키보드와 마우스 입력이 같은 프레임에 들어옴 | 마지막 입력을 우선한다 |

[확인 필요] 같은 프레임의 입력 충돌 처리 방식을 확정해야 한다.

---

# 9. 확인 필요 사항

- [확인 필요] 패드 조작을 프로토타입 범위에 넣을지 정해야 한다
- [확인 필요] 같은 프레임에서 키보드와 마우스 입력이 충돌할 때 우선순위를 정해야 한다
- [확인 필요] 행동 중 캐릭터 선택 시 표시할 상태 패널의 정보량을 정해야 한다
