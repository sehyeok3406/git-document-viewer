# 전투 프로토타입 범위 정의서

> 문서 상태: 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리  
> 목적: 이번 프로토타입에서 구현할 범위와 제외할 범위를 명확히 자른다

---

# 1. 개요

본 문서는 전투 프로토타입에서 무엇을 만들고, 무엇을 만들지 않을지 정의한다.  
프로토타입의 목적은 완성형 전투가 아니라, `실시간 명령`, `행동 시간`, `속성 약점`, `보스 기절`이 실제 플레이에서 이해되고 재미로 이어지는지 확인하는 것이다.  
따라서 구현 범위는 핵심 루프 검증에 필요한 요소로 제한한다.

---

# 2. 프로토타입 목표

전투 프로토타입은 아래 경험을 확인하는 것을 목표로 한다.

- 플레이어가 3명의 캐릭터 상태를 보고 명령을 내릴 수 있다
- 각 캐릭터는 명령 후 행동 시간이 지나야 행동을 발동한다
- 보스는 미공개 약점을 가지고 시작한다
- 플레이어는 속성 행동으로 약점을 찾고 반복 공략한다
- 공개 약점을 일정 횟수 공략하면 보스가 기절한다
- 기절은 플레이어에게 보상 타이밍으로 인식된다
- 전투 UI만 보고 현재 해야 할 행동을 이해할 수 있다

---

# 3. 포함 범위

## 3-1. 전투 구조

- 플레이어 캐릭터 3명
- 보스 1마리
- 단일 전투 1종
- 전투 시작, 전투 진행, 승리, 패배, 결과 표시

## 3-2. 캐릭터 상태

- 대기
- 명령 선택 중
- 행동 중
- 행동 발동
- 행동 불가 또는 전투 불능

## 3-3. 플레이어 행동

- 일반 공격
- 속성 공격 또는 속성 스킬
- 방어 또는 회복 행동 중 최소 1종
- 타겟 선택
- 행동 시간 표시

## 3-4. 보스 시스템

- 보스 HP
- 미공개 약점 슬롯
- 공개 약점 슬롯
- 약점 누적 공략 카운트
- 보스 기절 상태
- 간단한 자동 행동

## 3-5. UI

- 플레이어 캐릭터 3명의 상태 표시
- 현재 행동명과 남은 행동 시간 표시
- 보스 HP 표시
- 약점 슬롯 표시
- 기절 상태 표시
- 기본 명령 메뉴
- 전투 로그 또는 결과 메시지

---

# 4. 제외 범위

아래 항목은 이번 프로토타입의 기본 구현 범위에서 제외한다.

- 다수 일반 몬스터 전투
- 여러 보스와 보스별 상세 패턴
- 장비, 성장, 레벨업
- 인벤토리 전체 시스템
- 스토리 이벤트
- 세이브/로드
- 전투 외 탐험
- 복잡한 상태이상
- 속성 간 상성표
- 고급 AI
- 완성형 연출, 컷신, 사운드 연출
- 최종 아트 품질의 UI

필요하면 임시 연출과 더미 리소스는 사용할 수 있다. 단, 핵심 루프 검증을 흐릴 정도로 연출 구현에 시간을 쓰지 않는다.

---

# 5. MVP 완료 기준

아래 항목이 모두 가능하면 전투 프로토타입 MVP가 완료된 것으로 본다.

- 전투를 시작하면 플레이어 3명과 보스 1명이 화면에 표시된다
- 대기 중인 캐릭터에게 명령을 내릴 수 있다
- 명령을 받은 캐릭터는 일정 시간 후 행동을 발동한다
- 속성 행동이 보스 약점과 맞으면 약점이 공개되거나 누적된다
- 공개 약점을 정해진 횟수만큼 공략하면 보스가 기절한다
- 기절 중 보스 행동이 제한된다
- 보스 HP가 0이 되면 승리한다
- 플레이어 전원이 전투 불능이면 패배한다
- 플레이어가 UI를 보고 다음 명령 대상을 판단할 수 있다

---

# 6. 구현 우선순위

1. 전투 진입과 종료 흐름
2. 플레이어 3명과 보스 1명의 기본 상태 표시
3. 캐릭터별 대기/행동 상태 전환
4. 명령 메뉴와 행동 시간 처리
5. 일반 공격과 피해 처리
6. 속성 행동과 약점 판정
7. 약점 누적과 보스 기절
8. 보스 자동 행동
9. 최소 UI 정리
10. 테스트용 밸런스 조정

---

# 7. 확인 필요 사항

- [확인 필요] 프로토타입의 기본 입력 장치는 키보드, 마우스, 패드 중 무엇을 우선할지 정해야 한다
- [확인 필요] 보스 기절 중 약점 누적이 계속 가능한지 정해야 한다
- [확인 필요] 행동 중 피격 시 행동이 취소되는지 유지되는지 정해야 한다
- [확인 필요] 방어와 회복 중 어떤 행동을 먼저 넣을지 정해야 한다
- [확인 필요] 프로토타입 첫 전투의 목표 플레이 시간 범위를 정해야 한다
