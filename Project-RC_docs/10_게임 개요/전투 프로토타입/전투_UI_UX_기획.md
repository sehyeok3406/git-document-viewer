# 전투 UI/UX 기획서

> 문서 상태: 러프 초안  
> 기준 전투 모델: 플레이어 3인 파티 vs 보스 1마리

---

# 1. 개요

본 문서는 실시간 명령형 약점 공략 전투에서 필요한 UI/UX 정보를 정의한다.  
전투 UI는 플레이어 캐릭터 3명의 상태, 행동 시전 시간, 보스 약점, 약점 누적 공략, 보스 기절 상태를 명확하게 보여줘야 한다.  
현재 목표는 최종 UI 디자인이 아니라, 전투 이해에 필요한 정보 구조를 정리하는 것이다.  
본 문서는 시각적 스타일보다 정보 우선순위와 화면에서 반드시 보여줘야 할 요소를 중심으로 작성한다.

---

# 2. 기획 의도

이 전투는 여러 캐릭터의 행동 상태와 보스 약점 상태를 동시에 봐야 하므로 UI 가독성이 전투 이해도에 직접 영향을 준다.  
플레이어가 누가 대기 중인지, 누가 행동 중인지, 어떤 행동이 몇 초 뒤 발동되는지 즉시 알아야 한다.  
보스 약점은 전투의 핵심 목표이므로 미공개, 공개, 누적 공략, 비활성화 상태가 명확해야 한다.  
UI의 목적은 플레이어에게 정보를 과하게 쏟아내는 것이 아니라, 다음 판단에 필요한 정보만 빠르게 읽히게 하는 것이다.

---

# 3. 기획 방향성

UI는 캐릭터 상태와 보스 약점 정보를 가장 높은 우선순위로 둔다.  
플레이어가 명령을 내려야 하는 캐릭터는 즉시 눈에 띄어야 하며, 행동 중인 캐릭터는 타이머로 남은 시간을 보여줘야 한다.  
보스 약점 슬롯은 전투 목표를 나타내는 핵심 UI이므로 항상 확인 가능한 위치에 배치하는 것이 좋다.  
도트 게임 분위기를 고려하되, 정보 전달력이 떨어지는 장식은 최소화한다.

---

# 4. 시스템 규칙

## 4-1. 플레이어 캐릭터 UI

각 캐릭터 UI에는 캐릭터 이름, 현재 상태, 현재 행동명, 시전 시간 또는 남은 시간이 표시되어야 한다.  
대기 상태인 캐릭터는 명령 입력 가능 상태로 강조한다.  
행동 중인 캐릭터는 타이머 바를 통해 행동 발동까지 남은 시간을 보여준다.  
행동이 발동되면 상태 표시가 다시 대기로 전환되어야 한다.

## 4-2. 보스 약점 UI

보스 UI에는 미공개 약점 슬롯, 공개 약점 슬롯, 비활성화 약점 슬롯을 표시한다.  
미공개 약점은 `?` 또는 유사한 아이콘으로 표현한다.  
공개된 약점에는 속성 아이콘과 누적 공략 카운트를 함께 표시한다.  
비활성화된 약점은 흐리게 처리하거나 잠금 표시를 사용해 더 이상 공략 대상이 아님을 보여준다.

## 4-3. 기절 상태 UI

보스가 기절하면 화면에서 즉시 알 수 있는 강한 상태 표시가 필요하다.  
기절 중에는 보스의 행동 제한 상태와 남은 기절 시간을 보여주는 것이 좋다.  
기절이 종료되면 약점 갱신 연출을 통해 기존 약점 비활성화와 신규 약점 추가를 알려야 한다.  
기절 표시가 너무 약하면 플레이어가 보상 타이밍을 놓칠 수 있으므로 시각/사운드 피드백을 함께 고려한다.

---

# 5. 예외 케이스

세 캐릭터가 동시에 대기 상태가 되었을 때 UI가 과하게 점멸하거나 혼란스러워질 수 있다.  
여러 행동이 동시에 발동될 때 행동 결과와 약점 변화가 겹쳐 보이지 않도록 표시 우선순위가 필요하다.  
작은 화면이나 낮은 해상도에서 캐릭터 상태와 약점 슬롯을 모두 표시할 수 있는지 반응형 기준이 필요하다.  
기절, 약점 공개, 약점 비활성화가 짧은 시간에 연속 발생할 경우 연출이 겹치는 문제는 추후 검토해야 한다.

---

# 6. 관련 데이터

기획자가 조정해야 하는 UI 데이터는 상태명, 행동명, 속성 아이콘, 약점 슬롯 표시 방식, 기절 표시 문구, 타이머 바 표현 방식이다.  
시스템 내부에서 UI에 전달해야 하는 값은 캐릭터 상태, 현재 행동, 남은 시전 시간, 보스 약점 공개 여부, 누적 카운트, 기절 남은 시간이다.  
각 속성은 고유 아이콘 또는 색상 규칙이 필요하지만, 정확한 색상은 아트/UI 단계에서 확정한다.  
UI 애니메이션 타이밍과 사운드 피드백은 연출 문서에서 추가로 다룬다.

---

# 7. 타 시스템과의 연관성

실시간 명령 시스템은 캐릭터 상태와 시전 시간을 UI에 전달해야 한다.  
속성/약점/기절 시스템은 약점 슬롯, 누적 카운트, 기절 상태 변화를 UI에 전달해야 한다.  
보스전 구조와 몬스터 행동 시스템은 보스 행동 예고와 위험 정보를 UI에 표시할 수 있어야 한다.  
프로토타입 검증에서는 UI 없이 전투가 이해되는지가 아니라, 최소 UI를 붙였을 때 플레이어가 명령과 약점을 놓치지 않는지를 확인해야 한다.
